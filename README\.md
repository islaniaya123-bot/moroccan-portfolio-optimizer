#  BVC Portfolio Lab — Optimisation GARCH + Copules

Application Streamlit pour l'optimisation de portefeuille sur le marché marocain (BVC/MASI),
basée sur la méthodologie de Prince (2007) — HEC Montréal.

## Installation & lancement local

```bash
# 1. Cloner le dépôt
git clone https://github.com/VOTRE_USERNAME/bvc-portfolio-lab.git
cd bvc-portfolio-lab

# 2. Créer un environnement virtuel
python -m venv venv
source venv/bin/activate        # Linux/Mac
venv\Scripts\activate           # Windows

# 3. Installer les dépendances
pip install -r requirements.txt

# 4. Lancer l'application
streamlit run app.py
```

L'application s'ouvre automatiquement sur http://localhost:8501

##  Déploiement Streamlit Cloud (pour le QR code de la présentation)

1. Pousser ce dépôt sur GitHub (public)
2. Aller sur https://share.streamlit.io
3. Connecter votre compte GitHub → sélectionner ce repo → `app.py`
4. Cliquer **Deploy**
5. L'URL générée (ex: `https://bvc-portfolio.streamlit.app`) sert de QR code

##  Fonctionnalités

| Onglet | Contenu |
|--------|---------|
| Vue d'ensemble | KPIs, allocation optimale, poids en camembert et barres |
| GARCH & Volatilité | Variance conditionnelle simulée, paramètres estimés |
| Analyse Copule | Nuage U(0,1), heatmap corrélation, comparaison copules |
| Portefeuille Optimal | Évolution poids one-step-ahead, richesse cumulée, Sharpe glissant |
| Frontière Efficiente | Courbe risque/rendement, max Sharpe, min variance |
| Risque & CVaR | Distribution des pertes, VaR/CVaR 95%/99%, contribution au risque |
| Statistiques | Descriptives, kurtosis, test Jarque-Bera, ARCH, Q-Q plot |

##  Paramètres ajustables (barre latérale)

- **Actifs** : ATW, BCP, IAM, OCP, LHM, CIH, MNG, ADH, Bons du Trésor, MASI
- **Modèle GARCH** : GARCH(1,1) ou NGarch asymétrique (α, β, κ)
- **Copule** : Normale, Student-t, Clayton, Gumbel, Frank (ρ, ν, θ)
- **Mode d'optimisation** : Minimiser variance (rendement cible) ou maximiser rendement (variance cible)
- **Contraintes** : ventes à découvert autorisées/interdites
- **Simulation** : 500–10 000 trajectoires Monte Carlo, horizon 21–252 jours
